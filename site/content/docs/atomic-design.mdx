# Atomic Design Taxonomy

The Open edX Design System uses **atomic design** to classify every registered component
into one of five levels.

## Classification levels

| Level | Description | Examples |
|-------|-------------|---------|
| **atom** | Indivisible UI primitive | Button, Icon, Badge, Input, Label |
| **molecule** | Small group of atoms with a single purpose | SearchInput, FormGroup, AlertDismissible |
| **organism** | Complex, self-contained UI section | Navbar, CourseCard, DataTable |
| **template** | Page-level wireframe composed of organisms | CourseOutlineTemplate, DashboardTemplate |
| **page** | A fully realised route, usually a template + real data | LearnerDashboardPage, CourseHomePage |

## Usage in the registry

Every component MDX file must include an `atomicLevel` field in its frontmatter
matching one of the five values above (lowercase). The registry URL structure
mirrors this: `/registry/<atomicLevel>/<name>`.
