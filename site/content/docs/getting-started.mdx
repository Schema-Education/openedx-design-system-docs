## Overview (placeholder)

Contributing a component registration will involve:

1. **Identifying the component** in a source MFE (e.g., `frontend-app-learning`)
2. **Classifying it** using the [Atomic Design Taxonomy](/docs/atomic-design)
3. **Creating an MDX file** under `content/registry/<atomicLevel>/<ComponentName>.mdx`
   following the manifest schema defined in `registry/schema/component.schema.json`
4. **Running the validator** (`pnpm validate:registry`) to confirm the manifest is well-formed
5. **Opening a pull request** — the CI pipeline will lint, typecheck, and rebuild the index
